# 版本管理规范

## 概述

本文档定义了项目中依赖包版本管理的规范和最佳实践，确保项目的稳定性和可重现性。

## 版本固定原则

### 1. 强制使用精确版本

所有依赖包必须使用**精确版本号**，禁止使用版本范围前缀：

✅ **正确示例：**

```json
{
  "dependencies": {
    "@nestjs/core": "11.0.1",
    "rxjs": "7.8.1"
  }
}
```

❌ **错误示例：**

```json
{
  "dependencies": {
    "@nestjs/core": "^11.0.1",    // 使用了 ^ 前缀
    "rxjs": "~7.8.1",             // 使用了 ~ 前缀
    "lodash": "*",                // 使用了通配符
    "axios": "latest"             // 使用了 latest
  }
}
```

### 2. 禁止使用的版本格式

- `^1.2.3` - 兼容版本（允许升级到1.x.x的最新版本）
- `~1.2.3` - 补丁版本（允许升级到1.2.x的最新版本）
- `*` - 通配符版本
- `latest` - 最新版本
- `>=1.2.3` - 大于等于版本
- `>1.2.3` - 大于版本
- `<1.2.3` - 小于版本
- `<=1.2.3` - 小于等于版本
